Page 31. Module Overview
The Letter & Document Template Management Module provides schools with a centralized 
system for creating, managing, approving, and distributing official letters, notices, and 
documents. It integrates directly with existing system modules—Admissions, Discipline, 
Finance, Attendance, and Communication—to auto-populate recipient data and trigger context-
aware document generation.
1.1 Purpose
Liberian schools generate hundreds of official letters each academic year—from acceptance 
letters to disciplinary notices to fee reminders. Currently, most schools rely on manually typed 
Word documents or handwritten letters, leading to inconsistency, lost records, and no audit trail.  
This module solves these problems by providing:
•Standardized, school-branded letter templates
•Dynamic data population from the system database
•Role-based approval workflows before sensitive letters are sent
•Multi-channel delivery suited to Liberia’s infrastructure
•Complete audit trail for legal protection and accountability
1.2 Scope
This module covers all official correspondence generated by the school, organized into seven 
categories: Admissions, Disciplinary, Academic, Financial, Attendance, Communication, and 
Administrative. It does NOT cover informal messaging (handled by the Communication Hub) or 
WAEC-related correspondence (handled externally by WAEC).

5.2 PDF Download/Print (Primary Channel)
This is the default and most reliable delivery method for Liberia:
•Admin generates letter → system creates formatted PDF
•PDF includes school letterhead, formatting, and signature
•Bulk print option: generate PDFs for entire class/grade
•Print queue management for schools with shared printers
•Physical distribution via students or school messengers
5.3 In-App Notification
•Letter appears as notification in parent’s portal dashboard
•Read receipt: system tracks when parent opens the letter
•Parent can download PDF from the notification
•Push notification (PWA) alerts parent of new letter
5.4 SMS Link
•System sends SMS: “New letter from [School Name]. View: [short-link]”
•Short link opens mobile-optimized letter view (no login required for 24 hours)
•Secured with one-time token expiring after 24 hours
•Falls back to SMS summary if link cannot be opened
5.5 Email Attachment
•PDF attached to formatted email
•Email body contains letter summary and school contact info
•Available where parent has provided valid email address

Page 136.2 Approval Workflow Steps
1. Drafter creates or triggers letter generation
2. If approval required: letter enters “Pending Approval” status
3. Approver receives in-app notification with letter preview
4. Approver reviews and either Approves, Requests Changes, or Rejects
5. If approved: letter moves to “Ready to Send” queue
6. If changes requested: drafter receives feedback and revises
7. Approved letter can be printed, emailed, or sent via SMS link
6.3 Proprietor Review (NTR & Expulsion)
NTR (Never To Return) and Expulsion letters are the most consequential documents a school 
can issue. While the Proprietor role is read-only by design (per v4 ethics principle), these letters 
require a special “Proprietor Review” step:
•The Principal approves the letter
•The system automatically notifies the Proprietor
•The Proprietor can view the letter and add a comment (logged in audit trail)
•The letter is held for 24 hours before becoming sendable
•The Proprietor cannot block the letter (read-only principle) but their review is logged
This ensures the Proprietor is always informed of the most serious disciplinary actions without 
violating the read-only access principle.

7.1 Event-Driven Triggers
Letters can be triggered automatically by system events:
•Admission decision recorded → Acceptance/Rejection letter queued
•Disciplinary incident logged → Warning letter drafted (pending approval)
•Fee deadline approaching → Fee Reminder auto-sent (7 days before)
•Payment received → Payment Receipt auto-generated
•Absence threshold exceeded → Absenteeism notice drafted (pending approval)
•Term grades finalized → Promotion/Retention notices generated
•GPA threshold met → Honor Roll certificate queued
